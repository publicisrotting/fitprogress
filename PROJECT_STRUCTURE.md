# Полное объяснение структуры проекта FitProgress

Этот документ объясняет каждый файл и папку в проекте простыми словами. Я написал его так, чтобы даже человек, который ничего не понимает в кодинге, мог понять, за что отвечает каждый элемент. Теперь с ещё большим количеством деталей!

## 🏠 Корень проекта (главная папка)

### `package.json`
Это главный "реестр" всего проекта. Здесь написано:
- Как запускать проект (команды `npm run dev`, `npm run client`, `npm run server`, `npm run mobile`)
- Какие сторонние программы нужны для работы
- Как собирать проект для публикации
- Версия проекта, название, автор

Пример содержимого:
```json
{
  "name": "FitProgress Mobile App UI",
  "version": "0.1.0",
  "scripts": {
    "dev": "concurrently \"npm run client\" \"npm run server\"",
    "client": "vite --config frontend/vite.config.ts",
    "server": "npm run dev --prefix backend",
    "mobile": "npm start --prefix expo"
  }
}
```

### `package-lock.json`
Автоматически создаётся системой npm. Он фиксирует точные версии всех установленных библиотек, чтобы проект работал одинаково на разных компьютерах. Без него версии могут отличаться и что-то сломаться.

### `README.md`
Это документация проекта. Здесь обычно пишут:
- Как запускать проект
- Что делает проект
- Полезные заметки для разработчиков
- Как установить зависимости

### `.gitignore`
Список файлов, которые не нужно сохранять в систему контроля версий (Git). Например:
- `node_modules/` (библиотеки)
- `.env` (секреты)
- `build/` (готовый проект)
- `.DS_Store` (системные файлы Mac)

### `scripts/`
Папка с вспомогательными утилитами.
- `get-ip.js` - скрипт для получения IP-адреса компьютера
- `update-ip.js` - скрипт для обновления IP-адреса (для мобильной разработки)

### `build/`
Папка, куда складывается готовый собранный проект для публикации в интернете. Сюда попадают файлы, которые уже готовы для запуска на сервере. Создаётся командой `npm run build`.

### `node_modules/`
Это огромная папка с библиотеками, которые скачивает npm. Тут лежит код внешних программ, которые использует проект. Ты туда почти не заходишь - это не твой код. Устанавливается командой `npm install`.

## 🌐 `frontend/` - веб-приложение (то, что видит пользователь в браузере)

### `frontend/index.html`
Главная страница, которая загружает приложение. Это как каркас дома, в который ставится React-приложение. Содержит:
- `<div id="root"></div>` - место, куда React вставит интерфейс
- Мета-теги для мобильных устройств
- Ссылки на манифест и иконки для PWA

### `frontend/vite.config.ts`
Конфигурация сборщика Vite. Говорит системе:
- Как собирать проект
- Где искать файлы
- На каком порту запускать веб-сервер (5174)
- Как проксировать API-запросы на backend

### `frontend/tsconfig.json`
Настройки языка TypeScript. Это как правила, по которым пишется код в этой папке:
- Какие файлы компилировать
- Какие проверки типов включить
- Где искать типы

### `frontend/tsconfig.node.json`
Похожий файл, но для инструментов разработки на компьютере. Отличается настройками для Node.js окружения.

### `frontend/postcss.config.js`
Настройка обработки стилей. Важно для Tailwind CSS (системы стилей). Говорит, как преобразовывать CSS с помощью PostCSS.

### `frontend/public/`
Статические файлы, которые просто копируются в готовый проект:
- Картинки (favicon, иконки)
- `manifest.json` - для PWA (Progressive Web App)
- `sw.js` - service worker для кэширования
- Другие файлы, которые не нужно обрабатывать

## 📁 `frontend/src/` - основной код веб-приложения

### `frontend/src/main.tsx`
Стартовая точка приложения. Здесь:
- Импортируется React
- Подключается приложение к странице `index.html`
- Запускается приложение
- Подключаются провайдеры (Auth и Settings)
- Регистрируется service worker для PWA

### `frontend/src/App.tsx`
Главный файл приложения. Решает, какой экран показать:
- Если пользователь не авторизован → показывает `LoginScreen`
- Если авторизован → показывает основной интерфейс с навигацией
- Использует `useAuth()` для проверки статуса входа

### `frontend/src/config.ts`
Общие настройки. Здесь хранится адрес сервера (`API_URL`), чтобы фронтенд знал, куда отправлять запросы. Пример:
```typescript
export const API_URL = 'http://localhost:5001/api';
```

### `frontend/src/translations.ts`
Слова для разных языков. Содержит тексты интерфейса на русском, украинском, английском. Структура:
```typescript
export const translations = {
  en: { login: 'Login', ... },
  ru: { login: 'Войти', ... },
  uk: { login: 'Увійти', ... }
};
```

### `frontend/src/index.css`
Глобальные стили. Обычный CSS, который применяется ко всему приложению:
- Импорт шрифтов
- Базовые стили для body
- Переменные CSS для цветов

### `frontend/src/vite-env.d.ts`
Определения типов для Vite. Технический файл для TypeScript, который говорит компилятору о типах Vite.

## 🧠 `frontend/src/context/` - глобальные состояния приложения

### `frontend/src/context/AuthContext.tsx`
Логика авторизации. Хранит:
- Токен пользователя (для API-запросов)
- Информацию о входе (isAuthenticated)
- Функции входа/выхода (login, logout)
- Статус премиума (isPremium)
- Данные пользователя (user)

### `frontend/src/context/SettingsContext.tsx`
Настройки приложения. Управляет:
- Темой (светлая/тёмная) - меняет класс на body
- Языком - переключает слова в интерфейсе
- Единицами измерения (метры/футы)
- Переводом текста через функцию `t()`

## 🖥️ `backend/` - сервер (работает в фоне, обрабатывает данные)

### `backend/index.js`
Главный файл сервера. Делает:
- Запускает сервер на порту 5001
- Подключает базу данных MongoDB
- Включает защиту (helmet) от уязвимостей
- Подключает маршруты (routes)
- Парсит данные из запросов (JSON, формы)

### `backend/package.json`
Зависимости сервера. Содержит:
- Список нужных пакетов (express, mongoose, bcryptjs, etc.)
- Команду `npm run dev` для запуска с автоматической перезагрузкой (nodemon)

### `backend/.env`
Файл с секретами. Содержит:
- `MONO_MERCHANT_TOKEN` - токен для оплаты (Mono Bank)
- `JWT_SECRET` - ключ для JWT-токенов
- `MONGODB_URI` - URL базы данных
- `GOOGLE_CLIENT_ID` - для Google авторизации
- Этот файл НЕЛЬЗЯ публиковать в Git

## 🧱 `backend/models/` - описание данных в базе

### `backend/models/User.js`
Модель пользователя. Описывает, какие данные хранятся о каждом пользователе:
- `email` - уникальный email
- `password` - хэшированный пароль
- `googleId` - ID от Google (для OAuth)
- `name` - имя пользователя
- `picture` - URL аватара
- `stats` - статистика (тренировки, достижения, streak)
- `age`, `gender`, `weight`, `height`, `goal` - профиль
- `isPremium` - статус премиума
- `premiumExpiresAt` - дата окончания премиума

### `backend/models/SubscriptionInvoice.js`
Модель платежа. Хранит информацию о каждом платеже:
- `user` - ссылка на пользователя
- `planId` - ID плана ('premium_monthly')
- `amount` - сумма в копейках
- `ccy` - валюта (980 = UAH)
- `invoiceId` - ID счета в Mono Bank
- `pageUrl` - ссылка на оплату
- `status` - статус ('created', 'pending', 'success', 'failed')
- `createdAt`, `updatedAt` - timestamps

### `backend/models/Exercise.js`
Модель упражнения. Описывает упражнения в библиотеке:
- Название
- Описание
- Группа мышц
- Инструкции
- Видео

### `backend/models/Program.js`
Модель программы тренировок:
- Название программы
- Описание
- Уровень сложности
- Список упражнений
- Продолжительность

### `backend/models/Workout.js`
Модель тренировки пользователя:
- Дата тренировки
- Упражнения с весами/повторениями
- Время выполнения
- Калории

### `backend/models/Notification.js`
Модель уведомлений:
- Тип уведомления
- Текст
- Статус (прочитано/не прочитано)
- Дата

## 🚪 `backend/routes/` - API маршруты (пути сервера)

### `backend/routes/auth.js`
Авторизация. Обрабатывает:
- `POST /register` - регистрация нового пользователя
- `POST /login` - вход по email/паролю
- `POST /google` - вход через Google OAuth
- `GET /verify` - проверка токена
- `POST /logout` - выход

### `backend/routes/billing.js`
Оплата. Управляет:
- `POST /checkout` - создаёт счёт в Mono Bank
- `GET /latest` - получает последний счёт пользователя
- `GET /status?invoiceId=...` - проверяет статус оплаты
- `POST /cancel` - отменяет счёт
- `POST /downgrade` - отменяет премиум

### `backend/routes/user.js`
Пользователь. Работает с:
- `GET /profile` - получает данные профиля
- `PUT /profile` - обновляет профиль
- `GET /stats` - получает статистику

### `backend/routes/workouts.js`
Тренировки. Управляет:
- `GET /` - получает список тренировок
- `POST /` - создаёт новую тренировку
- `PUT /:id` - обновляет тренировку
- `DELETE /:id` - удаляет тренировку

### `backend/routes/exercises.js`
Упражнения. Работает с:
- `GET /` - получает список упражнений
- `GET /:id` - получает упражнение по ID
- `POST /search` - поиск упражнений

### `backend/routes/programs.js`
Программы тренировок:
- CRUD операции для программ

### `backend/routes/notifications.js`
Уведомления пользователя:
- Получение уведомлений
- Отметка как прочитанные

## 🔗 Как всё работает вместе (подробно)

1. **Пользователь открывает приложение:**
   - В браузере: `frontend/index.html` загружает `frontend/src/main.tsx`
   - На телефоне: `expo/App.js` открывает WebView с `frontend/`

2. **Пользователь взаимодействует:**
   - `frontend/src/components/screens/` рисует интерфейс
   - `frontend/src/context/AuthContext.tsx` хранит токен
   - `frontend/src/context/SettingsContext.tsx` управляет настройками
   - Запросы идут на `backend/routes/` через `frontend/src/config.ts`

3. **Сервер обрабатывает:**
   - `backend/index.js` получает запросы
   - `backend/routes/` обрабатывают логику
   - `backend/models/` работают с данными в MongoDB
   - Возвращает JSON-ответы

4. **Оплата работает через:**
   - `frontend/src/components/modals/PremiumModal.tsx` - кнопки и UI
   - `backend/routes/billing.js` - создание счёта в Mono Bank API
   - `backend/models/SubscriptionInvoice.js` - хранение данных о платеже
   - После оплаты Mono перенаправляет обратно в приложение

## 📌 Что важно знать для понимания

- **Если хочешь изменить экран или кнопки** → `frontend/src/components/screens/`
- **Если хочешь изменить логику оплаты** → `backend/routes/billing.js`
- **Если хочешь изменить данные пользователя** → `backend/models/User.js`
- **Если хочешь изменить мобильное поведение** → `expo/App.js`
- **Если хочешь добавить новый экран** → Создай файл в `frontend/src/components/screens/`
- **Если хочешь добавить новый API** → Создай маршрут в `backend/routes/`
- **Если хочешь изменить стили** → `frontend/src/index.css` или компоненты в `ui/`
- **Если хочешь добавить перевод** → `frontend/src/translations.ts`
